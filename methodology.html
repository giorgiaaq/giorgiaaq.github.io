Research Methodology

The methodology section describes the approach and processes used to investigate the Italian architectural heritage, focusing on the contributions of Frederick II.

The following steps outline the key phases of the research (click on the highlighted sections to know more):

  1. Exploration of the ArCo ontologies and knowledge graph through the ArCo SPARQL Query Editor ;
  2. Usage of SPARQL query language;
  3. Usage of LLMs to retrieve pieces of information to enrich the KG and prompting techniques to generate the best responses;
  4. Usage of GitHub to publish the website.

The methodological framework ensures a rigorous and systematic approach to data enrichment, contributing to the preservation and accessibility of Italy's cultural heritage.

ArCo SPARQL Query Editor

SPARQL Query Language

SPARQL (SPARQL Protocol and RDF Query Language) is a query language and protocol for querying and manipulating data stored in Resource Description Framework (RDF) format.

The Resource Description Framework (RDF) is a standard model for representing structured information about resources on the web. Developed by the World Wide Web Consortium (W3C), RDF provides a framework to describe and interlink data in a way that is both machine-readable and interoperable across different systems. RDF data is represented as triples, each triple consists of three components:

  1. Subject: The entity or resource being described.
  2. Predicate: The property or relationship between the subject and the object.
  3. Object: The value or another resource that is related to the subject by the predicate.

SPARQL queries generally consist of the following parts:

  1. PREFIX: Defines shorthand prefixes for long URIs to simplify the query.
  2. SELECT: Specifies the variables to return in the query result.
  3. WHERE: Contains the triple patterns that the query matches against the RDF data.
  4. FILTER: Applies conditions to restrict the results returned.
  5. ORDER BY: Sorts the results based on one or more variables.
  6. LIMIT: Restricts the number of results returned.
  7. UNION: Considers two or more patterns as valid.
  8. REGEX: Applies regular expressions matching to filter query results.

Large Language Models

Large Language Models (LLMs) are a class of artificial intelligence models designed to understand, generate, and manipulate human language. These models are typically based on deep learning techniques and are trained on vast amounts of text data.
LLMs used for this project:

ChatGPT

ChatGPT is a conversational AI model developed by OpenAI, based on the GPT (Generative Pretrained Transformer) architecture. It can understand and generate human-like text. The model has been trained on diverse text data, enabling it to hold conversations, generate content, and assist with various tasks.

Llama

LLaMA is a family of large language models developed by Meta (formerly Facebook). It is designed to be more efficient and accessible than other large models, focusing on providing high performance while using fewer computational resources. LLaMA models are trained on large datasets and are open-sourced.

Gemini

Gemini is a suite of AI models developed by Google DeepMind, designed to integrate the capabilities of large language models with more advanced reasoning and problem-solving abilities. It combines cutting-edge techniques to enhance the model's ability to understand and generate complex, context-aware responses across various domains.


Prompting techniques

Prompting techniques are methods used to elicit specific responses or behaviors from AI models, particularly language models. These techniques are crucial for guiding the AI to produce relevant and accurate outputs based on the provided inputs.
Some examples of the techniques used for the project:

  • Zero-shot prompting: Asking the model to perform a task without providing any specific examples or context. It relies on the model's general training to generate a response.
  • Few-shot prompting: Providing a few examples or context to guide the AI's responses. This helps the model understand the format or type of answer expected.
  • Multimodal Chain-Of-Thought: Integrating multiple types of data (text, images, audio) to enhance reasoning.
